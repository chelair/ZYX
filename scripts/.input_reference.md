## 续算逻辑

### 续算目录结构

```
{project_path}/{sub_dir}/      ← 原始目录
                    ├── con1/  ← 第一次续算（在原始目录内）
                    ├── con2/  ← 第二次续算（与 con1 同级，非 con1/con2）
                    ├── con3/  ← 以此类推
                    ├── INCAR
                    ├── CONTCAR
                    └── ...
```

### 结构优化中断续算（job_continue.py）

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 确定当前工作目录 | 原始目录 或 最新 conN 目录（均在原始目录内同级排列） |
| 2 | N = max(con1,con2,...) + 1 | 新续算编号 |
| 3 | `mkdir {sub_dir}/con{N}/` | 新建同级续算目录（无 -p） |
| 4 | `cp -n` INCAR、KPOINTS、POTCAR、CONTCAR → con{N}/ | 安全复制，不覆盖 |
| 5 | `mv` WAVECAR → con{N}/ | **仅当 OUTCAR 有完成标志时** |
| 6 | 节点检查 + 队列选择 | SSH 运行 bhosts / bqueues / blimits |
| 7 | 生成 vasp.lsf | -q 用推荐队列，-J 用 jobname |
| 8 | 写入 `job.info` | 项目/子任务/队列/参数/job_id 记录 |
| 9 | `bsub < vasp.lsf` | 提交作业 |

### 节点检查与队列选择

提交流程中自动执行以下检查（复用 `jobs_monitor.py` 的队列推荐逻辑 `recommend_queue()`）：

```
第1步: bhosts         ← 逐节点检查空闲核
  目标: 找 ≥2 个各有 ≥24 空闲核的节点（你的 job 需要 48核，ptile=24）

第2步: bqueues        ← 队列 PEND/RUN 拥堵情况
  目标: 在能放下 job 的队列中选 PEND 最少的

```

示例分析（2026-06-23）：

| 队列 | bhosts (有无节点可放?) | bqueues | blimits | 结论 |
|------|----------------------|---------|---------|:----:|
| ocean6226R_1day | ✅ hd038/hd042/hd043/hd044 各≥24空闲 | PEND 336 | — | **推荐** |
| ocean_530_1day | ❌ 无单节点≥24空闲（散在 67核） | PEND 24 | — | 放不下 |
| normal_1day_new | ❌ 无单节点≥24空闲 + 8619 PEND | PEND 8619 | — | 不可用 |
| charge | ✅ 12节点全空 384核 | PEND 0 | — | 付费备选 |

默认 48 核（2 节点 × 24 核），对应 `#BSUB -n 48` + `#BSUB -R "span[ptile=24]"`。
### 作业命名规范

`#BSUB -J` 的 jobname 直接标识任务归属，方便 `bjobs` 快速识别：

```
格式: {项目名}_{子目录末段}
示例:
  a_Fe2O3_0701_Fe2O3_110
  a_Fe2O3_0701_2_1
  Co4N_0703_model1

规则:
  - 仅使用字母、数字、下划线 _（不用中文、空格、特殊符号）
  - 项目名取自 vaspcheck_projects.json 中的 name 字段
  - 子目录末段取自 sub.dir 的最后一级（去掉 opt/abs/ 等前缀）
```

### 状态变化追踪（渲染触发条件）

结构图只渲染 **近两次检查之间状态发生变化** 的任务，避免重复渲染。

#### 追踪机制

```
每次 vasp_check 运行流程:
                              ┌──────────────────┐
  ① 检查前                     │ 读取 .last_status │  ← 上次检查时的状态快照
                              └──────┬───────────┘
                                     ↓
  ② 执行检查                   更新 subtask.status
  ③ 检查后                     ┌──────────────────┐
                              │ 对比新旧状态       │
                              │ 找出 changed_list │
                              └──────┬───────────┘
                                     ↓
                              ┌──────────────────┐
  ④ 写回快照                    │ 保存当前状态      │
                              │ → .last_status   │
                              └──────┬───────────┘
                                     ↓
  ⑤ 生成PPT                    只渲染 changed_list 中的任务
```

#### 状态快照文件

位置：`{LOCAL_BASE}/.last_status.json`（与汇总报告同级）

```json
{
  "a_Fe2O3_0701|结构优化-110面": "Completed",
  "a_Fe2O3_0701|吸附-110_CH_1": "Run",
  "Co4N_0703|结构优化-模型1": "Stop"
}
```

键名格式：`{项目名}|{子任务名}`，用 `|` 分隔避免歧义。

#### changed_list 判定逻辑

```python
# 伪代码
old_statuses = load(".last_status.json")  # 可能不存在（首次运行）
new_statuses = {}                          # 所有子任务的当前状态

changed = []
for each project, subtask:
    key = f"{project}|{subtask}"
    new_statuses[key] = subtask.status
    old = old_statuses.get(key)
    if old is None:
        continue  # 新出现的子任务，不渲染（还没跑过）
    if old != subtask.status:
        changed.append((project, subtask))

save(".last_status.json", new_statuses)
```

| 当前状态 | 是否渲染 | 原因 |
|---------|:--------:|------|
| 状态 = Run（运行中） | ✅ **每次必渲染** | 监控是否发散/异常 |
| 状态变了（任何方向） | ✅ 渲染 | 新鲜完成或失败 |
| 状态没变 + 非 Run | ❌ 不渲染 | 上次已处理过 |
| 首次运行（无历史） | ❌ 全不渲染 | 无对比基准 |
| 新增子任务 | ❌ 不渲染 | 尚未跑过 |

#### 渲染时不检查本地 POSCAR

`_find_struct_dirs` 不再要求本地 `POSCAR` 文件存在。渲染所需的文件直接取自 `vasp_check` 刚下载到 `LOCAL_BASE/{project}/{subtask}/` 的结果。
### 提交标识文件 `job.info`

> **skill 识别作业的首要依据**，优先级高于 `vaspcheck_projects.json`。

每次提交作业时，在 con{N}/ 下生成一个 `job.info` 文件：

```ini
[submit]
project     = a_Fe2O3_0701          # 项目名
subtask     = opt/Fe2O3_110
directory   = opt/Fe2O3_110/con3    # 远程相对路径
queue       = ocean6226R_1day       # 提交队列
job_name    = a_Fe2O3_0701_Fe2O3_110  # bjobs 作业名
cores       = 48                    # 核数
status      = submitted              # submitted / running / completed / failed
submitted_at = 2026-06-23 21:00:00  # 提交时间
job_id      = 123456                # bsub 返回后回填

[incar]
IBRION      = 2
NSW         = 500
ENCUT       = 500
ISPIN       = 2
IVDW        = 11
```

**使用场景：**
- `job_continue.py` 读取 `job.info` 确定续算来源
- `vasp_check.py` 优先用 `job.info` 定位项目（回退到 JSON）
- `jobs_monitor.py` 匹配作业时优先用 `job.info` 的 project/subtask
- 用户 `cat job.info` 一眼识别目录归属

**写入方式：** `safe_write_text(client, remote_path, content)` 原子写入，不覆盖

## 作业核数估算规则

根据体系规模和计算类型，合理分配核数以平衡效率和资源。

### 结构优化 (IBRION=2)

| 体系规模 | 默认核数 | 说明 |
|---------|:--------:|------|
| 总原子数 < 100 | **24** (1节点) | 默认参数，单节点足够 |
| 100 ≤ 原子数 ≤ 150 | **24~36** | 可适当增加 |
| 原子数 > 150 | **48** (2节点×24核) | 默认 |

| 特殊条件 | 调整 | 原因 |
|---------|:----:|------|
| DFT+U 体系 | 在原基础上增加 | U 值计算需要更多电子步迭代 |
| 含大量稀土元素 | 在原基础上增加 | 稀土元素波函数更复杂，对角化更慢 |


### 电子结构（静态自洽 / DOS / 能带）

| 条件 | 推荐核数 | 说明 |
|------|:--------:|------|
| 默认参数 | **24** | 单节点，无特殊要求 |
| ISYM = -1 / 0 或 LREAL = .FALSE. | **≥ 36** | 对称性关闭或实空间投影关闭后计算量增大 50%~100% |


### HSE 混合泛函

> 经验基准：**每 24 核 × 24 小时 ≈ 10 个电子步**

估算公式：

```
预计耗时 = (电子步数 / 10) × 24 小时 × (24 / 核数)
```

示例：系统需 60 电子步收敛，用 48 核：
```
预计耗时 = (60 / 10) × 24 × (24 / 48) = 72 小时 = 3 天
```


### NEB（过渡态）

| 推荐核数 | 说明 |
|:--------:|------|
| **≥ 48** (2节点) | NEB 需要同时优化多个镜像，核数不足→单个离子步极慢 |

> NEB 核数还需满足 `核数 ÷ IMAGES = 整数`，以确保核在各镜像间均匀分配。

### 参数判断要点

| 参数 | 需要调整的情况 |
|------|---------------|
| POTIM | 含 H 等轻原子 → 降低至 **1.0** |
| NSW | 需要更长轨迹 → 增大；仅平衡 → 500-1000 即可 |
| TEBEG/TEEND | 退火模拟：TEEND < TEBEG（如 TEBEG=3000, TEEND=300）；若为恒定高温运行则保持相等 |
| SMASS | 温度波动剧烈 → 增大 (2~3)；温度响应太慢 → 减小 (0.5) |
| NELMDL | 结构接近收敛 → 用 -10 足够；初始结构远离平衡 → 增大到 -20~-30 |

! 第二段: TEBEG=2000, TEEND=300,  NSW=2000  (退火降温)

## VASPKit 用法

路径: `/data/gpfs03/mdye/vaspkit.1.3.5/bin/vaspkit`

### 402 - Fix Selected Atoms（固定原子）

用于 slab 计算时固定底部原子层，上层和吸附物自由弛豫。

```
运行: vaspkit → 4 (Structure Editor)
            → 1 (POSCAR) / 2 (CONTCAR)
            → 402 (Fix Selected Atoms)
            → 固定模式:
                1 - 按原子序号/元素 (Fix by Atomic Indices/Labels)
                2 - 按层数 (Fix by Layers)
                3 - 按高度 (Fix by Heights)
                4 - 从 SELECTED_ATOMS_LIST 文件读
```

#### 固定规则（按场景）

> ⚠️ 判断依据：**`job.info` / `vaspcheck_projects.json` 中的 subtask 名称**，不通过 POSCAR 内容猜测。

| 场景 | 固定方式 | 固定比例 | 判断依据 |
|------|---------|:--------:|---------|
| **slab 初始优化** | 3) Fix by Heights | 底部 ~40% | 子任务名含"结构优化"、"opt"、"raw"、"slab" |
| **吸附构型** | 3) Fix by Heights | 底部 ~50% | 子任务名含"吸附"、"abs"、"OER"、"反应" |
| **频率矫正** | 1) Fix by Indices | 除吸附质外的**所有原子** | 子任务名含"频率"、"freq"、"振动" |

#### 固定比例计算

```
z 范围 = z_max - z_min (Direct 坐标)
阈值 = z_min + 比例 × (z_max - z_min)
固定: z ≤ 阈值 → F F F
自由: z > 阈值 → T T T

slab 优化: 比例 = 0.40
吸附构型: 比例 = 0.50
```

#### 频率矫正特殊说明

频率矫正时，**除了吸附质分子外，slab 的所有原子都应该被固定**。
此时不能按高度固定（会固定掉 slab 底部但吸附质所在的顶部也会动），
必须采用模式 1 按原子序号固定：

```
1. 读取 POSCAR 确定哪些原子是吸附质，哪些是 slab 基底
2. 吸附质通常位于 slab 表面，原子数少，元素与 slab 不同
3. 若无法判断 → 告知用户人工操作
4. 固定所有非吸附质原子: 402 → 1 → 输入基底原子序号范围
```

#### 操作示例

```bash
# slab 优化: 按高度固定底部 40%
vaspkit → 4 → 1 → 402 → 3  → 输入 z 阈值
# 吸附构型: 按高度固定底部 50%
vaspkit → 4 → 1 → 402 → 3  → 输入 z 阈值
# 频率矫正: 按原子序号固定基底
vaspkit → 4 → 1 → 402 → 1  → 输入基底原子序号 (如 "1-123")
```

结果：POSCAR 自动添加 `Selective Dynamics`，固定原子标 `F F F`，自由原子标 `T T T`。

## 偶极矫正（吸附构型）

吸附计算时 slab 上下表面不对称会产生偶极矩，需加偶极矫正。

### 参数设置

```fortran
LDIPOL = .TRUE.          ! 开启偶极矫正
IDIPOL = 3               ! 真空层沿 c 轴方向
DIPOL  = 0.5 0.5 {z}    ! {z} = 原子沿真空层方向的几何中心 (Direct 坐标)
```

### z 中心计算方法

```
1. 读取 POSCAR 找出真空层方向（通常 c 轴）
2. 提取所有原子的 Direct z 坐标
3. 几何中心 = (z_min + z_max) / 2
4. DIPOL = 0.5 0.5 {几何中心保留两位小数}
   例: 0.225471 → 0.23
```

### INCAR 规则

- **无中文注释**：INCAR 中不得出现中文，注释用英文或不用
- **参数对齐**：`关键字 = 值` 格式，等号两侧空格可选

### 判断依据

| 类型 | 需要偶极矫正? |
|------|:------------:|
| 吸附（slab 不对称） | ✅ 是 |
| 体相计算 | ❌ 否 |
| 分子/团簇 | ❌ 否 |
| 纯 slab（两面相同） | ❌ 否 |

## KPOINTS 模板

### 标准模板

```fortran
auto kppoints
0
G
1 1 1
0 0 0
```

- 第 1 行：注释（忽略）
- 第 2 行：`0` 表示自动生成 k 点网格
- 第 3 行：`G` Gamma 中心网格（金属体系推荐）；`M` 为 Monkhorst-Pack
- 第 4 行：k 点网格 `N1 N2 N3`，需根据晶格常数计算（见下方）
- 第 5 行：位移（Gamma 中心网格通常为 `0 0 0`）

### k 点确定规则

> **k × 晶格常数 > 20 Å**

即每个方向上的 k 点数满足：

```
Ni = ceil(20 / ai)
```

其中 ai 为该方向的晶格常数 (Å)，Ni 向上取整到最近的整数。

| 晶格常数 (Å) | 最少 k 点数 | 常用取值 |
|:-------------:|:----------:|:---------:|
| 3 | ceil(20/3) = 7 | 7×7×7 |
| 5 | ceil(20/5) = 4 | 4×4×4 |
| 8 | ceil(20/8) = 3 | 3×3×3 |
| 10 | ceil(20/10)= 2 | 2×2×2 |
| 20 | ceil(20/20)= 1 | 1×1×1 |
| 非等轴晶胞 | 各方向独立计算 | 如 5×5×3 |

示例：对于晶格常数 a=b=5.0Å，c=8.0Å 的正交晶胞：
```
auto kppoints
0
G
5 5 3
0 0 0
```

集成在 `vasp_check.py` 的 `check_subtask` 中，每个子任务检查完后自动诊断。

### 检测项清单

| 检测项 | 代码 | 严重度 | 数据来源 |
|--------|------|:------:|---------|
| SCF 不收敛 | SCF_NOT_CONVERGED | 🟡 | OSZICAR 电子步数 |
| 离子步未收敛 | IONIC_NOT_CONVERGED | 🟡 | OUTCAR 收敛标志 + 力 |

| 能量震荡 | ENERGY_OSCILLATION | 🟡 | OSZICAR dE 符号交替 |

## 提交脚本

### 主脚本: vasp.lsf（LSF 集群）

```bash
bsub < submit/vasp.lsf
```

| 参数 | 值 | 说明 |
|------|----|------|
| 队列 | normal_1day_new | 稳定队列，低挂起风险 |
| 核数 | 48 (2节点×24核) | span[ptile=24] |
| Walltime | 24:00 | 24 小时 |
| 预告 | -wt 50 -wa URG | 到期前 50s 发 SIGURG |
| VASP 路径 | `/data/gpfs03/mdye/vasp.6.4.2/bin/vasp_std` | 标准版 |
| MPI | I_MPI_FABRICS=shm:ofi | 共享内存 + OFI |
| 输出 | %J.out / %J.err（LSF 日志）+ result（运行日志） | |

#### Walltime 优雅退出机制

`lsf_watcher` 后台进程捕获 `SIGURG`（LSF 到期前 50 秒发出）→ 写入 `STOPCAR`（内含 `LSTOP = .TRUE.`）→ VASP 检测到后在本离子步结束时**优雅退出**，保证 CONTCAR 完整。

续算时会 `rm -rf STOPCAR` 清除上次的 STOPCAR。
